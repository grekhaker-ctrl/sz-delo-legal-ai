# ⚖️ СЗ ДЕЛО | ЮРИДИЧЕСКИЙ ИИ-АГЕНТ

**Профессиональная система для юридического отдела строительной компании**

---

## 🎯 НАЗНАЧЕНИЕ

Автоматизированный ИИ-помощник для юристов компании «СЗ Дело», работающей в сфере строительства жилых комплексов в Москве и Московской области.

Система помогает:
- ✅ Анализировать договоры на риски
- ✅ Готовить юридические заключения
- ✅ Заполнять шаблоны договоров
- ✅ Сравнивать версии документов
- ✅ Отвечать на юридические вопросы
- ✅ Искать в базах Гарант/КонсультантПлюс

---

## 🚀 БЫСТРЫЙ СТАРТ

### 1. Установка зависимостей

```bash
# Python 3.11+
pip install -r requirements.txt
```

### 2. Настройка

```bash
# Скопируйте конфигурационный файл
cp .env.example .env

# Отредактируйте .env и добавьте ключи API
nano .env
```

### 3. Запуск

```bash
# Запуск Streamlit приложения
streamlit run app.py
```

**Откроется в браузере:** http://localhost:8501

## 📋 ФУНКЦИОНАЛ

### 1. 💬 Чат с ИИ-юристом

**Что умеет:**
- Отвечать на юридические вопросы
- Консультировать по договорам
- Анализировать судебную практику
- Давать рекомендации по строительным договорам

**Примеры вопросов:**
- «Что такое субподряд и какие риски для генподрядчика?»
- «Можно ли прописать неустойку 1% в день за просрочку?»
- «Кто отвечает за скрытые дефекты после сдачи дома?»

**Важно:** Бот отвечает **только на юридические вопросы**!

---

### 2. 📄 Анализ рисков договора

**Возможности:**
- Автоматический анализ загруженного договора
- Выявление юридических рисков
- Проверка на соответствие ГК РФ
- Рекомендации по исправлению
- Оценка риска по шкале 0-10

**Находит:**
- 🔴 Чрезмерную неустойку
- 🔴 Неограниченную ответственность
- 🟠 Односторонние условия
- 🟠 Отсутствие порядка приёмки
- 🟡 Неопределённые сроки
- 🟡 Отсутствие гарантий

**Результат:**
- Список рисков с цветовыми индикаторами
- Предлагаемые формулировки для исправления
- Итоговое заключение (подписывать/не подписывать)

---

### 3. ⚖️ Юридическое заключение

**Что делает:**
- Определяет тип договора
- Анализирует с точки зрения интересов СЗ Дело
- Даёт заключение: ✅/⚠️/❌
- Список ключевых причин
- Рекомендации по изменениям

---

### 4. 📝 Заполнить шаблон

**Как работает:**
1. Выберите шаблон из списка
2. Заполните реквизиты в форме
3. Система автоматически заполнит договор
4. Скачайте готовый файл DOCX

**Доступные шаблоны:**
- Договор строительного подряда
- Договор субподряда
- Договор поставки материалов
- Договор аренды спецтехники
- Договор на проектные работы

---

### 5. 🔄 Сравнить версии

**Возможности:**
- Загрузка двух версий договора
- Визуальное сравнение (diff с подсветкой)
- Анализ юридических последствий изменений
- Рекомендации: принять/отклонить правки

**Пример использования:**
- Контрагент прислал свою версию договора
- Нужно понять, что изменилось и чем это рискованно

## 🏗️ ТЕХНИЧЕСКАЯ ИНФОРМАЦИЯ

### Стек технологий

| Компонент | Технология |

| **Веб-интерфейс** | Streamlit |

## 🔧 НАСТРОЙКА LLM

### YandexGPT (основной)

**Получение ключей:**

1. Создайте аккаунт в [Yandex Cloud](https://cloud.yandex.ru/)
2. Создайте платёжный аккаунт
3. Создайте сервисный аккаунт с ролью `ai.languageModels.user`
4. Создайте API-ключ для сервисного аккаунта
5. Скопируйте `folder_id` и `api_key` в `.env`

**Стоимость:** ~0.5₽ за 1000 токенов

### GigaChat (резервный)

**Получение ключей:**

1. Зарегистрируйтесь в [Sber ID](https://id.sber.ru/)
2. Создайте приложение в [GigaChat Developer](https://developers.sber.ru/)
3. Получите `client_id` и `client_secret`
4. Создайте credentials: `base64(client_id:client_secret)`
5. Скопируйте в `.env`

**Бесплатно** в тестовом режиме

## 📝 ПРИМЕРЫ ИСПОЛЬЗОВАНИЯ

### Пример 1: Анализ договора подряда

**Сценарий:** Контрагент прислал договор на 20 страниц

**Действия:**
1. Откройте раздел "📄 Анализ рисков договора"
2. Загрузите файл (PDF или DOCX)
3. Нажмите "Начать анализ рисков"
4. Получите отчёт:
   - ✅ Можно подписывать / ⚠️ С правками / ❌ Не рекомендуется
   - Список найденных рисков
   - Рекомендуемые формулировки

**Результат:** Экономия 2-3 часов работы юриста

---

### Пример 2: Заполнение шаблона

**Сценарий:** Нужно быстро подготовить договор субподряда

**Действия:**
1. Откройте раздел "📝 Заполнить шаблон"
2. Выберите "Договор субподряда"
3. Заполните реквизиты:
   - Номер и дата договора
   - Заказчик и Подрядчик (название, ИНН)
   - Адрес объекта
   - Цена и описание работ
4. Нажмите "Заполнить шаблон"
5. Скачайте готовый DOCX

**Результат:** Договор готов за 2 минуты вместо 30 минут

---

### Пример 3: Сравнение версий

**Сценарий:** Контрагент вернул договор со своими правками

**Действия:**
1. Откройте "🔄 Сравнить версии"
2. Загрузите:
   - Версия 1: ваша исходная
   - Версия 2: версия контрагента
3. Нажмите "Сравнить версии"
4. Получите:
   - Визуальный diff с подсветкой
   - Список изменений
   - Юридический анализ последствий

**Результат:** Понимаете все риски за 5 минут

## ⚠️ ВАЖНЫЕ ЗАМЕЧАНИЯ

### 1. Только юридические вопросы

Бот **отказывается** отвечать на неюридические вопросы:
- ❌ «Какая погода в Москве?»

Все сервисы доступны из РФ:
- ✅ YandexGPT — российские серверы

## 🐛 РЕШЕНИЕ ПРОБЛЕМ

### Ошибка: "LLM недоступен"

**Причина:** Не настроены ключи API

**Решение:**
1. Проверьте `.env` файл
2. Убедитесь что `YANDEX_FOLDER_ID` и `YANDEX_API_KEY` заполнены
3. Перезапустите приложение

### Ошибка: "Не могу прочитать файл"

**Причина:** Файл повреждён или формат не поддерживается

**Решение:**
1. Проверьте формат (PDF, DOCX, TXT)
2. Убедитесь что файл не защищён паролем
3. Попробуйте конвертировать в PDF

### Ошибка: "Слишком большой файл"

**Причина:** Файл больше 10 MB

**Решение:**
1. Сожмите PDF
2. Разделите на части
3. Удалите лишние приложения
