## 🖥️ ЛОКАЛЬНЫЙ ЗАПУСК

**Для:** Тестирования и демонстрации

### Требования:
- Python 3.11+
- 4 GB RAM
- 2 GB дискового пространства

### Установка:

```bash
# Клонирование репозитория
git clone <repo-url>
cd sz_delo_legal_ai

# Создание виртуального окружения
python -m venv venv
venv\Scripts\activate  # Windows
source venv/bin/activate  # Linux/Mac

# Установка зависимостей
pip install -r requirements.txt

# Копирование конфига
copy .env.example .env  # Windows
cp .env.example .env  # Linux/Mac

# Настройка .env (откройте в редакторе)
notepad .env  # Windows
nano .env  # Linux/Mac
```

### Запуск:

```bash
streamlit run app.py
```

**Доступ:** http://localhost:8501

## 🐳 DOCKER

**Для:** Продакшена, простота развёртывания

### Требования:
- Docker 20+
- Docker Compose 2+

### Файл Dockerfile:

Создайте файл `Dockerfile`:

```dockerfile
FROM python:3.11-slim

# Установка системных зависимостей
RUN apt-get update && apt-get install -y \
    tesseract-ocr \
    tesseract-ocr-rus \
    libtesseract-dev \
    libleptonica-dev \
    && rm -rf /var/lib/apt/lists/*

# Рабочая директория
WORKDIR /app

# Копирование зависимостей
COPY requirements.txt .

# Установка Python зависимостей
RUN pip install --no-cache-dir -r requirements.txt

# Копирование приложения
COPY . .

# Создание директорий
RUN mkdir -p data/templates data/sz_delo_base data/cache

# Переменные окружения
ENV PYTHONUNBUFFERED=1
ENV STREAMLIT_SERVER_PORT=8501
ENV STREAMLIT_SERVER_ADDRESS=0.0.0.0

# Порт
EXPOSE 8501

# Команда запуска
CMD ["streamlit", "run", "app.py"]
```

### Файл docker-compose.yml:

Создайте файл `docker-compose.yml`:

```yaml
version: '3.8'

services:
  legal-ai:
    build: .
    ports:
      - "8501:8501"
    environment:
      - YANDEX_FOLDER_ID=${YANDEX_FOLDER_ID}
      - YANDEX_API_KEY=${YANDEX_API_KEY}
      - GIGACHAT_CREDENTIALS=${GIGACHAT_CREDENTIALS}
    volumes:
      - ./data/templates:/app/data/templates
      - ./data/sz_delo_base:/app/data/sz_delo_base
      - ./data/cache:/app/data/cache
    restart: unless-stopped
```

### Запуск:

```bash
# Создание .env файла
cp .env.example .env

# Редактирование .env
nano .env

# Сборка и запуск
docker-compose up -d

# Просмотр логов
docker-compose logs -f

# Остановка
docker-compose down
```

**Доступ:** http://localhost:8501

## 🖧 VPS (Ubuntu 22.04)

**Для:** Постоянного использования в компании

### Требования:
- 2 CPU cores
- 4 GB RAM
- 40 GB SSD
- Ubuntu 22.04

### Шаг 1: Подготовка сервера

```bash
# Обновление системы
sudo apt update && sudo apt upgrade -y

# Установка зависимостей
sudo apt install -y python3-pip python3-venv git nginx

# Установка Tesseract OCR
sudo apt install -y tesseract-ocr tesseract-ocr-rus

# Создание пользователя
sudo adduser legalai
sudo usermod -aG sudo legalai
```

### Шаг 2: Установка приложения

```bash
# Переключение на пользователя
su - legalai

# Клонирование репозитория
git clone <repo-url> /home/legalai/sz_delo_legal_ai
cd /home/legalai/sz_delo_legal_ai

# Создание виртуального окружения
python3 -m venv venv
source venv/bin/activate

# Установка зависимостей
pip install -r requirements.txt

# Копирование конфига
cp .env.example .env
nano .env  # Заполните ключи API
```

### Шаг 3: Systemd сервис

```bash
sudo nano /etc/systemd/system/legal-ai.service
```

Вставьте:

```ini
[Unit]
Description=SZ Delo Legal AI Agent
After=network.target

[Service]
User=legalai
Group=legalai
WorkingDirectory=/home/legalai/sz_delo_legal_ai
Environment="PATH=/home/legalai/sz_delo_legal_ai/venv/bin"
ExecStart=/home/legalai/sz_delo_legal_ai/venv/bin/streamlit run app.py --server.address 0.0.0.0 --server.port 8501
Restart=always

[Install]
WantedBy=multi-user.target
```

Запуск:

```bash
sudo systemctl daemon-reload
sudo systemctl enable legal-ai
sudo systemctl start legal-ai
sudo systemctl status legal-ai
```

### Шаг 4: Nginx (reverse proxy)

```bash
sudo nano /etc/nginx/sites-available/legal-ai
```

Вставьте:

```nginx
server {
    listen 80;
    server_name legal-ai.sz-delo.ru;  # Ваш домен

    location / {
        proxy_pass http://127.0.0.1:8501;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # Для WebSocket (Streamlit)
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

Активация:

```bash
sudo ln -s /etc/nginx/sites-available/legal-ai /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

### Шаг 5: HTTPS (SSL сертификат)

```bash
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d legal-ai.sz-delo.ru
```

**Доступ:** https://legal-ai.sz-delo.ru

## 🏢 КОРПОРАТИВНЫЙ СЕРВЕР

**Для:** Развёртывания в инфраструктуре компании

### Требования:
- Доступ к корпоративной сети
- Корпоративный LDAP/AD (опционально)
- Корпоративный прокси (если есть)

### Настройка прокси:

```bash
# Если нужен прокси для доступа к интернету
export HTTP_PROXY=http://proxy.sz-delo.ru:8080
export HTTPS_PROXY=http://proxy.sz-delo.ru:8080

# Добавьте в ~/.bashrc для постоянного применения
echo "export HTTP_PROXY=http://proxy.sz-delo.ru:8080" >> ~/.bashrc
echo "export HTTPS_PROXY=http://proxy.sz-delo.ru:8080" >> ~/.bashrc
```

### Интеграция с LDAP/AD:

Для корпоративной аутентификации используйте:

```bash
pip install streamlit-authenticator
```

Создайте файл `authentication.py`:

```python
import streamlit_authenticator as stauth

# Конфигурация аутентификации
config = {
    'credentials': {
        'usernames': {
            'ivanov': {
                'email': 'ivanov@sz-delo.ru',
                'name': 'Иванов И.И.',
                'password': 'hashed_password'  # Используйте hash
            }
        }
    },
    'cookie': {
        'expiry_days': 30,
        'key': 'random_key',
        'name': 'legal_ai_cookie'
    }
}

authenticator = stauth.Authenticate(config)
```
